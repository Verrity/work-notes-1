Типы портов:
1. `Access`
	1. Исходящий трафик (от коммутатора к ПК). 
		1. `vlan 5 => untagged`
		2. `untagged => <drop>`
	2. Входящий трафик (от ПК в коммутатор)
		1. `vlan 5 => <drop>`
		2. `untagged => vlan 5`
2. `Trunk`
	1. Пропускает только тегированый трафик
	2. Если указан `native vlan`, то заключает весь нетегированый трафик в `native vlan`. В исходящем трафике метка снимается, в исходящем добавляется.
3. General
	1. Иходящий трафик
		1. `network X` помечен как `tagged` Уходит с тегом X
		2. `network Y`помечен как `untagged`Снять тег → уходит чистым (untagged)
		3. network Z` не разрешен на порту DROP  (даже не подойдет к выходу)
		4. Если указано mark as `tagged`: Кадр VLAN 5 => **СНИМАЕТ ТЕГ** → уходит **untagged**
		5. Если указано mark as `untagged`:  Кадр VLAN 5 => **ОСТАВЛЯЕТ ТЕГ** → уходит **tagged 10**
	2. Входящий трафик
		1. `tagged X`Если VLAN X разрешен как `tagged` Пропустить, тег оставить
		2. `tagged X`Если VLAN X разрешен только как `untagged` или вообще не разрешен `DROP`
		3. `untagged`Если есть VLAN, назначенный как `untagged` (обычно один) Добавить тег этого VLAN
		4. `untagged`Если нет VLAN для `untagged` `DROP`
		5. Если указано mark as `tagged`/`untagged`;
			1. Пришло untagged - не попадает в vlan 5 автоматически, для этого нужна отдельная команда `switchport general pvid 5`. 
			2. Пришло `tagged 5` - Пропускается, потому что VLAN 5 разрешен на порту
