У FreeRADIUS нет одного конфига настроек, как у других программ. Вместо этого он использует концепцию **виртуальных серверов**.
* `/etc/raddb/sites-available/` — это папка, где хранятся файлы конфигурации всех возможных виртуальных серверов
* `/etc/raddb/sites-enabled/` — это папка, где лежат символические ссылки (ярлыки) на файлы из `sites-available`. Сервер FreeRADIUS читает только эту папку! Если здесь есть ссылка на файл — виртуальный сервер работает

Так можно создать разные виртуальные сервера для разных условий.

* **`/etc/raddb/radiusd.conf`** - Главный конфигурационный файл. В нем подключаются модули, задаются глобальные настройки (права на запуск, пути к логам) и, самое важное, прописывается, где лежат виртуальные серверы
* **`/etc/raddb/mods-available/` и `mods-enabled/`** - Точно такая же система, как для виртуальных серверов, но для модулей. Сам модуль `sql`, `ldap`, `eap` настраивается в файле из `mods-available`, а включается ссылкой из `mods-enabled`. Без этого модуль будет просто неактивен.
* **`/etc/raddb/clients.conf`** — Святая святых! Здесь прописываются все ваши NAS (точки доступа, коммутаторы, VPN-серверы) и их общие секреты (secret). Если NAS не описан в этом файле, FreeRADIUS просто выбросит его пакет.
* **`/etc/raddb/users`** — Простейшая плоская база данных пользователей. Для тестов и очень маленьких инсталляций.
* **/etc/raddb/dictionary** — «Словарь» всех атрибутов RADIUS в человеко-понятном виде. Благодаря ему вы пишете в конфиге `Framed-IP-Address`, а сервер понимает, что это атрибут номер `8`.
